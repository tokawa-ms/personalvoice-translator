## 概要

Personal Voice Translator は、Azure Speech Service を使用したリアルタイム音声翻訳アプリケーションです。マイクから入力された音声を認識し、指定した言語に翻訳して、Personal Voice で音声合成します。

### 主な機能

- 🎤 **リアルタイム音声認識**: マイク入力をリアルタイムで認識
- 🌐 **自動翻訳**: 10言語間での自動翻訳
- 🔊 **Personal Voice 音声合成**: カスタム音声での読み上げ
- 💬 **チャット形式の UI**: 元の言語と翻訳結果を吹き出し表示
- 💾 **設定の永続化**: ブラウザーに設定を保存

## 必要な準備

### 1. Azure Speech Service リソースの作成

1. **Azure ポータルにアクセス**
   - https://portal.azure.com にサインイン

2. **Speech Service リソースを作成**
   - 「リソースの作成」→「AI + Machine Learning」→「Speech」を選択
   - 以下の情報を入力:
     - **サブスクリプション**: 使用する Azure サブスクリプション
     - **リソースグループ**: 新規作成または既存のものを選択
     - **リージョン**: `japaneast`（日本東部）または任意のリージョン
     - **名前**: 任意のリソース名
     - **価格レベル**: `F0`（無料）または `S0`（標準）

3. **キーとエンドポイントを取得**
   - リソース作成後、「キーとエンドポイント」セクションに移動
   - **キー1** または **キー2** をコピー（サブスクリプションキー）
   - **場所/リージョン**をメモ（例: `japaneast`）

### 2. Personal Voice の準備（オプション）

Personal Voice を使用する場合:

1. Azure Speech Studio にアクセス: https://speech.microsoft.com/
2. 「Custom Voice」セクションで Personal Voice を作成
3. Personal Voice ID を取得

**注**: Personal Voice を使用しない場合は、標準の音声が使用されます。

### 3. ブラウザーの準備

- **推奨ブラウザー**:
  - Google Chrome 90+
  - Microsoft Edge 90+
  - Firefox 88+
  - Safari 14+

- **マイク権限**:
  - ブラウザーがマイクへのアクセス許可を求めた場合は「許可」を選択

## 初期設定

### 1. アプリケーションを開く

1. `src/index.html` をブラウザーで開く
2. 設定モーダルが自動的に表示されます

### 2. Azure Speech Service を設定

**設定モーダル**で以下の情報を入力:

#### 必須項目

| 項目 | 説明 | 例 |
|------|------|-----|
| **サブスクリプションキー** | Azure Speech Service のキー | `a1b2c3d4e5f6...` |
| **リージョン** | リソースのリージョン | `japaneast` |

#### オプション項目

| 項目 | 説明 | デフォルト値 |
|------|------|-------------|
| **Personal Voice ID** | カスタム音声のID | （空白）標準音声を使用 |
| **音声名** | 使用する音声の名前 | `ja-JP-NanamiNeural` |
| **認識元言語** | マイク入力の言語 | `ja-JP`（日本語） |

### 3. 設定を保存

1. すべての必須項目を入力
2. 「保存」ボタンをクリック
3. 設定がブラウザーのローカルストレージに保存されます
4. サービスの初期化が完了すると、ステータスが「準備完了」になります

## 基本的な使い方

### 翻訳の開始

1. **翻訳先言語を選択**
   - ドロップダウンから翻訳先の言語を選択
   - 選択肢: 日本語、英語、中国語、韓国語、スペイン語、フランス語、ドイツ語、イタリア語、ポルトガル語、ロシア語

2. **翻訳を開始**
   - 「翻訳を開始」ボタンをクリック
   - ステータスが「翻訳中...」に変わります
   - マイク権限を求められた場合は「許可」を選択

3. **音声を入力**
   - マイクに向かって話す
   - 認識された音声が画面に表示されます
   - 翻訳結果が自動的に表示され、音声合成されます

4. **翻訳を停止**
   - 「翻訳を停止」ボタンをクリック

#### 吹き出しの色

- 🟣 **紫色（左側）**: 認識された原文
- 🟢 **緑色（右側）**: 翻訳されたテキスト
- 🔴 **赤色（中央）**: エラーメッセージ
- ⚪ **グレー（中央）**: システムメッセージ

## トラブルシューティング

### 音声が認識されない

**原因と対処法**:

1. **マイク権限が許可されていない**
   - ブラウザーの設定でマイク権限を確認
   - Chrome: `chrome://settings/content/microphone`
   - アドレスバーの🔒マークから権限を変更

2. **マイクが接続されていない**
   - マイクが正しく接続されているか確認
   - システムの音声設定でマイクを選択

3. **音声が小さい**
   - マイクに近づいて話す
   - システムのマイク音量を上げる

### 翻訳されない

**原因と対処法**:

1. **サブスクリプションキーが無効**
   - Azure ポータルでキーを確認
   - 設定を再入力して保存

2. **リージョンが間違っている**
   - Azure リソースのリージョンと一致しているか確認
   - 設定で正しいリージョンを入力

3. **ネットワーク接続の問題**
   - インターネット接続を確認
   - ファイアウォールが Azure への接続をブロックしていないか確認

### 音声合成されない

**原因と対処法**:

1. **スピーカー/ヘッドフォンの問題**
   - オーディオデバイスが接続されているか確認
   - システムの音量を確認

2. **音声名が無効**
   - 設定の「音声名」を確認
   - 標準の音声名（例: `ja-JP-NanamiNeural`）を使用

3. **Personal Voice ID の問題**
   - Personal Voice ID が正しいか確認
   - ID を空白にして標準音声を試す

### エラーメッセージ

| エラー | 原因 | 対処法 |
|--------|------|--------|
| `初期化エラー: サブスクリプションキーまたはリージョンが設定されていません` | 必須項目が未入力 | 設定で必須項目を入力 |
| `認識エラー: Invalid subscription key` | 無効なキー | Azure ポータルで正しいキーを確認 |
| `音声合成エラー: ...` | 音声合成の問題 | 音声名を確認、または Personal Voice ID を削除 |
| `Azure Speech SDK が読み込まれていません` | SDK の読み込み失敗 | ページをリロード、インターネット接続を確認 |

## よくある質問

### Q1. Personal Voice なしで使用できますか？

**A**: はい、使用できます。Personal Voice ID を空白にすると、Azure の標準音声が使用されます。

### Q2. どの言語がサポートされていますか？

**A**: 以下の10言語がサポートされています:
- 日本語 (ja-JP)
- 英語 (en-US)
- 中国語 (zh-CN)
- 韓国語 (ko-KR)
- スペイン語 (es-ES)
- フランス語 (fr-FR)
- ドイツ語 (de-DE)
- イタリア語 (it-IT)
- ポルトガル語 (pt-BR)
- ロシア語 (ru-RU)

### Q3. オフラインで使用できますか？

**A**: いいえ、このアプリケーションは Azure Speech Service との通信が必要なため、インターネット接続が必須です。

### Q4. 料金はかかりますか？

**A**: Azure Speech Service の使用料金が発生します。詳細は Azure の価格ページをご確認ください。無料枠（F0）も利用可能です。

### Q5. 複数の言語を同時に翻訳できますか？

**A**: 現在のバージョンでは、1つの翻訳先言語のみサポートしています。翻訳先言語はドロップダウンから変更できます。

### Q6. 翻訳履歴を保存できますか？

**A**: 現在のバージョンでは、翻訳履歴はブラウザーを閉じると消去されます。将来のバージョンで履歴保存機能を追加予定です。

### Q7. モバイルデバイスで使用できますか？

**A**: はい、モバイルブラウザーでも動作しますが、デスクトップブラウザーでの使用を推奨します。

### Q8. セキュリティは大丈夫ですか？

**A**: 
- サブスクリプションキーはブラウザーのローカルストレージに保存されます
- 通信は HTTPS で暗号化されます
- 音声データは Azure に送信されますが、保存はされません（設定による）

### Q9. カスタム音声（Personal Voice）の作成方法は？

**A**: 
1. Azure Speech Studio にアクセス: https://speech.microsoft.com/
2. 「Custom Voice」→「Personal Voice」を選択
3. 音声サンプルをアップロードして Personal Voice を作成
4. 作成された Personal Voice ID をアプリの設定に入力

### Q10. 設定を削除するには？

**A**: 設定モーダルを開き、「クリア」ボタンをクリックすると、すべての設定が削除されます。
