## 修正されたバグの概要

このリリースでは、以下の3つの重要なバグを修正しました：

1. 設定モーダルがマウスフォーカス外で意図せず閉じてしまう問題
2. Personal Voice ID を指定してもデフォルト音声で再生される問題
3. 翻訳先言語を指定してもすべて日本語で音声合成される問題

## Bug 1: 設定モーダルの予期しない閉じる動作

### 問題の詳細

**症状**:
- 設定画面（モーダル）で情報を入力中、マウスのフォーカスがモーダル外に移動すると、設定画面が自動的に閉じてしまう
- 入力途中のデータが失われる可能性がある
- ユーザーが「保存」ボタンを押す前に設定画面が閉じてしまう

**影響**:
- ユーザビリティの低下
- データ入力のやり直しが必要になる
- ユーザーの操作ミスを誘発

### 修正内容

**変更ファイル**: `src/js/app.js`

**変更箇所**:
- モーダル外クリックで閉じるイベントリスナーをコメントアウト（行102-108）
- Escキーで閉じるイベントリスナーをコメントアウト（行110-116）

**修正後の動作**:
- 設定モーダルは以下の方法でのみ閉じるようになりました：
  1. 「保存」ボタンをクリックして設定を保存した後
  2. 「×閉じる」ボタンを明示的にクリック
- モーダル外のクリックやEscキー押下では閉じなくなりました

## Bug 2: Personal Voice ID が適用されない問題

### 問題の詳細

**症状**:
- 設定画面で Personal Voice ID を入力しても、音声合成時にデフォルトの音声で再生される
- Personal Voice の特徴が反映されない
- カスタム音声が使用されない

**原因**:
- `speechSynthesis.js` の `synthesize` メソッドが常に `speakTextAsync` を使用していた
- Personal Voice を使用するには SSML（Speech Synthesis Markup Language）と `mstts:ttsembedding` タグが必要
- Personal Voice ID の設定が音声合成処理に反映されていなかった

### 修正内容

**変更ファイル**: `src/js/speechSynthesis.js`

**変更箇所**:
1. `synthesize` メソッドを修正（行134-181）
   - Personal Voice ID の有無をチェック
   - Personal Voice が設定されている場合は SSML を生成して `synthesizeSSML` を呼び出し
   - 標準音声の場合は従来通り `speakTextAsync` を使用

2. `generateSSML` メソッドを改善（行239-272）
   - Personal Voice ID が設定されている場合、`mstts:ttsembedding` タグを使用
   - `speakerProfileId` 属性に Personal Voice ID を設定

**修正後の動作**:
- Personal Voice ID が設定されている場合、SSML を使用して音声合成
- `mstts:ttsembedding` タグで Personal Voice の特徴を適用
- Personal Voice ID が未設定の場合は標準音声を使用

### コード差分

```javascript
// 修正前
async synthesize(text) {
    // ... 省略 ...
    return new Promise((resolve, reject) => {
        this.synthesizer.speakTextAsync(
            text,
            (result) => { /* ... */ },
            (error) => { /* ... */ }
        );
    });
}

// 修正後
async synthesize(text) {
    // ... 省略 ...
    const settings = window.stateManager.getState('settings');
    
    // Personal Voice が設定されている場合は SSML を使用
    if (settings && settings.personalVoiceId && settings.personalVoiceId.trim() !== '') {
        console.log('[SpeechSynthesis] Personal Voice を使用して SSML で合成');
        const ssml = this.generateSSML(text, settings);
        return await this.synthesizeSSML(ssml);
    } else {
        console.log('[SpeechSynthesis] 標準音声で合成');
        return new Promise((resolve, reject) => {
            this.synthesizer.speakTextAsync(/* ... */);
        });
    }
}
```

**SSML の例**:
```xml
<speak version="1.0" 
       xmlns="http://www.w3.org/2001/10/synthesis" 
       xmlns:mstts="https://www.w3.org/2001/mstts" 
       xml:lang="en-US">
  <voice name="en-US-JennyNeural">
    <mstts:ttsembedding speakerProfileId="YOUR_PERSONAL_VOICE_ID">
      Hello, this is synthesized with Personal Voice.
    </mstts:ttsembedding>
  </voice>
</speak>
```

## Bug 3: 翻訳先言語が音声合成に反映されない問題

### 問題の詳細

**症状**:
- 翻訳先言語を英語（en-US）に設定しても、日本語の音声で合成される
- 他の言語に設定しても、常に日本語の音声（ja-JP-NanamiNeural）で再生される
- 翻訳先のテキストと音声の言語が一致しない

**原因**:
1. 音声名が翻訳先言語に応じて自動的に変更されていなかった
2. 設定画面で指定した音声名（デフォルト: `ja-JP-NanamiNeural`）が常に使用されていた
3. 翻訳先言語を変更しても音声合成サービスが再初期化されなかった

### 修正内容

**変更ファイル**: `src/js/speechSynthesis.js`, `src/js/app.js`

#### 1. 言語ごとのデフォルト音声マッピングを追加

**新規メソッド**: `getVoiceNameForLanguage` （行274-311）

翻訳先言語コードに基づいて適切な音声名を自動選択:

| 言語コード | デフォルト音声名 | 音声の性別 |
|-----------|----------------|-----------|
| ja-JP | ja-JP-NanamiNeural | 女性 |
| en-US | en-US-JennyNeural | 女性 |
| zh-CN | zh-CN-XiaoxiaoNeural | 女性 |
| ko-KR | ko-KR-SunHiNeural | 女性 |
| es-ES | es-ES-ElviraNeural | 女性 |
| fr-FR | fr-FR-DeniseNeural | 女性 |
| de-DE | de-DE-KatjaNeural | 女性 |
| it-IT | it-IT-ElsaNeural | 女性 |
| pt-BR | pt-BR-FranciscaNeural | 女性 |
| ru-RU | ru-RU-SvetlanaNeural | 女性 |

**ロジック**:
1. カスタム音声名が設定されている場合
   - その音声名が翻訳先言語コードで始まる場合は、カスタム音声を使用
   - 例: 翻訳先が `en-US` でカスタム音声が `en-US-AriaNeural` の場合は `en-US-AriaNeural` を使用
2. カスタム音声名が未設定または言語が一致しない場合
   - デフォルト音声マッピングから自動選択

#### 2. 音声合成サービスの初期化を改善

**変更箇所**: `initialize` メソッド（行19-77）

```javascript
// 修正前
if (settings.voiceName) {
    speechConfig.speechSynthesisVoiceName = settings.voiceName;
}

// 修正後
const voiceName = this.getVoiceNameForLanguage(settings.targetLanguage, settings.voiceName);
speechConfig.speechSynthesisVoiceName = voiceName;
console.log('[SpeechSynthesis] 音声名:', voiceName);
console.log('[SpeechSynthesis] 対象言語:', settings.targetLanguage);
```

#### 3. 翻訳先言語変更時の再初期化

**変更ファイル**: `src/js/app.js`  
**変更箇所**: 翻訳先言語変更イベントリスナー（行92-112）

```javascript
// 修正後
window.uiManager.elements.targetLanguage.addEventListener('change', async (e) => {
    console.log('[AppController] 翻訳先言語変更:', e.target.value);
    const settings = window.stateManager.getState('settings');
    if (settings) {
        settings.targetLanguage = e.target.value;
        window.storageManager.updateSetting('targetLanguage', e.target.value);
        window.stateManager.updateSettings(settings);
        
        // 音声合成サービスを再初期化して、新しい言語の音声を使用
        if (window.stateManager.getState('isConnected')) {
            console.log('[AppController] 翻訳先言語変更により音声合成サービスを再初期化');
            try {
                await window.speechSynthesisService.initialize(settings);
                console.log('[AppController] 音声合成サービス再初期化完了');
            } catch (error) {
                console.error('[AppController] 音声合成サービス再初期化エラー:', error);
            }
        }
    }
});
```

**修正後の動作**:
1. 翻訳先言語を変更すると、その言語に適した音声が自動的に選択される
2. 音声合成サービスが再初期化され、新しい音声設定が適用される
3. 翻訳結果が適切な言語の音声で読み上げられる

### 使用例

**シナリオ 1**: 日本語から英語への翻訳
- 認識元言語: `ja-JP`（日本語）
- 翻訳先言語: `en-US`（英語）
- 使用される音声: `en-US-JennyNeural`（英語の女性音声）

**シナリオ 2**: 日本語から中国語への翻訳
- 認識元言語: `ja-JP`（日本語）
- 翻訳先言語: `zh-CN`（中国語）
- 使用される音声: `zh-CN-XiaoxiaoNeural`（中国語の女性音声）

**シナリオ 3**: カスタム音声の使用
- 認識元言語: `ja-JP`
- 翻訳先言語: `en-US`
- カスタム音声設定: `en-US-GuyNeural`（英語の男性音声）
- 使用される音声: `en-US-GuyNeural`（カスタム音声が優先）
